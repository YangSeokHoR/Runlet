# Runlet — 개발 지침서

> **가칭 안내:** 앱 이름 `Runlet`은 가칭이다. 언제든 교체 가능하다. (예시 실행 대상인 `KeyFlow`는 이 앱이 아니라, 이 런처가 실행시켜줄 별개의 Mac 앱이다.)
>
> **이 문서의 위상:** claude.ai code 프로젝트의 **기준 문서**다. 아래 "확정된 핵심 명세"는 임의로 변경하지 않는다. 바꿀 이유가 보이면 트레이드오프와 함께 합의를 먼저 받는다.

---

## 1. 프로젝트 개요

**한 줄 요약**
터미널에서 명령어를 쳐야만 실행되던 작업들을, 메뉴바 드롭다운의 버튼 하나로 실행하는 Mac 유틸리티.

**목적**
반복적으로 Terminal을 열고 → 디렉토리 이동하고 → 긴 명령어를 입력하는 과정을 없앤다. 자주 쓰는 명령(예: `KeyFlow` 빌드+실행)을 등록해두고, 메뉴바에서 한 번 클릭으로 실행한다.

**가치 가설**
- 개인이 반복하는 셸 작업은 3~10개 안팎으로 정해져 있다.
- 그 명령들을 "이름 붙은 버튼"으로 바꿔두면, 터미널을 여는 마찰이 사라진다.
- 특히 `xcodebuild` 같은 다단계 빌드 스크립트는 손으로 치기 번거롭기 때문에 버튼화의 효용이 크다.

## 2. 확정된 핵심 명세

| 항목 | 확정 내용 |
|---|---|
| 앱 성격 | 개인용 명령어 런처 (범용) |
| 앱 껍데기 | **메뉴바 앱** — 상단 메뉴바 아이콘 클릭 시 명령어 리스트 드롭다운 |
| 명령 관리 | 명령어를 **여러 개 등록/수정/삭제** 가능 |
| 실행 방식 | 버튼 클릭 시 **백그라운드에서 조용히 실행** (별도 터미널 창 안 뜸) |
| 실행 후 피드백 | **없음** (무피드백, 조용히 실행만) |
| 등록/수정 위치 | **별도 관리 창**에서 편집. 드롭다운은 **실행 전용** |
| 명령 데이터 모델 | **이름 + 여러 줄 셸 스크립트** (작업 디렉토리는 스크립트 내 `cd`로 처리) |
| 실행 엔진 | `Process`로 **로그인 셸(`/bin/zsh -l -c`)** 에 스크립트 전체를 전달 |
| 샌드박스 | **OFF** (임의 셸/바이너리 실행이 필요하므로 필수) |
| 배포 형태 | 개인용 로컬 앱 (Mac App Store 배포 아님) |
| 플랫폼 최소 버전 | macOS 13 (Ventura) 이상 — `MenuBarExtra` 사용 |

### 협업자에게 위임한 열린 결정

아래는 **아직 확정하지 않은** 결정이다. 합리적 기본값으로 진행하되, 바꾸려면 합의를 받는다.

| 열린 결정 | 임시 기본값 | 비고 / 리스크 |
|---|---|---|
| **오래 걸리는 명령의 실행 중 표시** | 없음 (확정된 무피드백 유지) | ⚠️ `xcodebuild`는 수십 초 소요 → "도는 중인지" 알 수 없음. 나중에 "실행 중 아이콘 살짝 변경" 정도의 **최소 표시**를 넣을지 합의 필요 |
| **같은 명령 중복 실행 처리** | 허용 (막지 않음) | 무피드백을 문자 그대로 지키기 위해 허용. 빌드 중복 실행이 문제되면 "실행 중 재클릭 무시"로 변경 가능 (단, 무피드백이면 사용자가 무시된 걸 모름 → 위 항목과 함께 결정) |
| **작업 디렉토리(cwd) 별도 필드** | 없음 (스크립트 내 `cd` 사용) | 예시가 `cd`를 직접 쓰므로 불필요. 나중에 편의상 cwd 입력칸을 추가할 수 있음 |
| **로그인 시 자동 실행** | 미포함 (M5에서 옵션 검토) | `SMAppService`로 추가 가능 |
| **명령 항목 아이콘/정렬** | 없음 (등록 순서대로) | 개수가 늘면 순서 변경·아이콘 추가 검토 |
| 앱 최종 이름 | `Runlet` (가칭) | 교체 가능 |

## 3. 비목표 (범위 밖)

- 터미널 에뮬레이터 기능 (명령 결과를 실시간 콘솔로 보여주는 것) — 이번 범위 아님.
- 명령 실행 히스토리/로그 뷰어 — 이번 범위 아님.
- 원격 서버 SSH 실행, 스케줄링(cron 유사) — 범위 아님.
- 여러 사용자 계정/동기화 — 범위 아님 (전적으로 이 Mac 로컬 개인용).
- Mac App Store 배포 및 샌드박스 대응 — 범위 아님 (샌드박스 OFF 전제).

## 4. 기술 스택 & 환경

**스택**
- 언어: Swift
- UI: SwiftUI (메뉴바 껍데기 `MenuBarExtra`, 관리 창)
- 셸 실행: Foundation `Process`
- 영속화: `Codable` + JSON 파일
- (선택, M5) 로그인 항목: `ServiceManagement` `SMAppService`

**권한 · 샌드박스 제약 메모 (중요)**
- **App Sandbox는 반드시 OFF.** 샌드박스가 켜져 있으면 `Process`로 임의의 셸/바이너리를 실행할 수 없다. 이 앱의 핵심 동작 자체가 막히므로 비-샌드박스가 전제다.
- 비-샌드박스이므로 `~/Library/Application Support/...` 등 파일 접근은 자유롭다.
- 로컬 개인용이라 Developer ID 서명·공증(notarization)은 필수 아님. Xcode에서 직접 실행하거나 `.app`을 복사해 쓰면 된다. 다른 곳에서 받은 것으로 처리되어 Gatekeeper 경고가 뜨면 최초 1회 우클릭 → 열기로 통과한다.
- **PATH 주의:** GUI 앱이 실행하는 프로세스는 사용자의 셸 PATH를 자동 상속하지 않는다. 그래서 셸을 `-l`(로그인 셸)로 실행해 `.zprofile`/`.zshrc`의 PATH(예: Homebrew `/opt/homebrew/bin`)를 물려받게 한다. `xcodebuild`는 `/usr/bin`에 있어 대개 문제없지만, Homebrew 도구를 쓰는 명령을 위해 로그인 셸이 안전하다.

## 5. 사용자 플로우

**A. 명령 실행 (주 사용 흐름)**
1. 메뉴바의 Runlet 아이콘을 클릭한다.
2. 드롭다운에 등록된 명령 리스트가 이름으로 나열된다.
3. 원하는 명령을 클릭한다.
4. 드롭다운이 닫히고, 명령이 백그라운드에서 조용히 실행된다. (별도 창·알림 없음)

**B. 명령 관리 (편집 흐름)**
1. 메뉴바 드롭다운 하단의 "명령어 관리…" 항목을 클릭한다.
2. 별도 관리 창이 열린다.
3. 관리 창에서 명령을 **추가 / 수정 / 삭제**한다. 각 명령은 `이름`과 `스크립트(여러 줄)`로 구성.
4. 변경 즉시 저장되고, 다음에 드롭다운을 열면 반영되어 있다.

## 6. 아키텍처 — 모듈 구조와 의존 방향

레이어를 셋으로 나눈다. 의존 방향은 **위 → 아래 한 방향**이다 (UI가 서비스에 의존, 서비스가 모델에 의존. 역방향 의존 없음).

```
[UI 레이어]  MenuBarExtra 리스트 뷰 · 관리 창 뷰
                     │  (호출)
                     ▼
[서비스 레이어]  CommandStore (목록 상태·저장/로드)
                 CommandRunner (셸 실행)
                     │  (사용)
                     ▼
[모델 레이어]   RunCommand (이름 + 스크립트 + id)
```

**구성 요소 (가칭)**
- `RunletApp` — 앱 진입점. `MenuBarExtra` scene과 관리 창(Window) scene 구성.
- `CommandStore` — 관찰 가능한 상태 객체. 명령 목록을 메모리에 들고, JSON으로 저장/로드. 추가·수정·삭제 API 제공.
- `CommandRunner` — 스크립트 문자열을 받아 `Process`로 실행하는 순수 실행기. UI를 모른다.
- `MenuBarListView` — 드롭다운. 명령 목록을 버튼으로 렌더링 + "명령어 관리…" 진입점.
- `ManagerWindowView` — 관리 창. 목록 편집 UI.

**의존 규칙**
- `CommandRunner`는 UI·저장소를 몰라야 한다 (스크립트 문자열만 받아 실행).
- `CommandStore`는 실행 로직을 몰라야 한다 (데이터만 관리).
- 뷰가 이 둘을 조합한다.

## 7. 데이터 모델

**`RunCommand`** — 명령 하나를 나타내는 값 타입.

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | 고유 식별자 | 항목 구분용 (자동 생성) |
| `name` | 문자열 | 드롭다운에 표시될 이름 (예: "KeyFlow 빌드+실행") |
| `script` | 문자열(여러 줄) | 실제 실행될 셸 스크립트 전체 |

- `Codable`로 직렬화하여 JSON 저장.
- 작업 디렉토리는 별도 필드 없이 `script` 안의 `cd`로 처리한다 (위임된 열린 결정 참고).

**영속화**
- 저장 위치: `~/Library/Application Support/Runlet/commands.json` (가칭 경로).
- 앱 시작 시 로드, 변경 시 저장.

**샘플 데이터 (첫 명령, 실사용 예시)**
```
이름:   KeyFlow 빌드+실행
스크립트:
cd ~/Developer/KeyFlow
./make-app.sh
open /Applications/KeyFlow.app
```

## 8. 핵심 기술 챌린지 & 해결 방향

막히기 쉬운 지점과 대응책이다. (구현자는 실제 코드 작성 전 각 API의 정확한 시그니처를 Apple 공식 문서로 확인한다.)

**8.1 여러 줄 · 셸 문법 스크립트 실행 — 가장 중요**
- 문제: 명령이 `$(...)`, 파이프, `awk`, `cd`, 따옴표를 포함한 여러 줄 스크립트다. 인자를 쪼개 바이너리를 직접 exec하면 셸 문법이 해석되지 않아 실패한다.
- 해결: `Process`의 실행 파일을 `/bin/zsh`로 두고, 인자로 `["-l", "-c", 스크립트전체문자열]`을 넘긴다. 이러면 스크립트가 실제 zsh로 해석되어 파이프·치환이 정상 동작한다.

**8.2 PATH / 환경 변수 상속**
- 문제: GUI에서 띄운 프로세스는 최소 PATH만 가져 Homebrew 도구 등을 못 찾을 수 있다.
- 해결: 위처럼 **로그인 셸(`-l`)** 로 실행해 사용자 셸 초기화 파일의 PATH를 상속. `xcodebuild`는 `/usr/bin` 고정이라 대개 안전하지만 로그인 셸이 범용적으로 안전하다.

**8.3 App Sandbox 충돌**
- 문제: 샌드박스가 켜지면 임의 셸 실행이 막혀 앱이 무력화된다.
- 해결: 샌드박스 OFF로 확정 (섹션 4). 프로젝트 설정에서 Sandbox capability를 넣지 않는다.

**8.4 오래 걸리는 명령 + 무피드백의 긴장**
- 문제: 빌드는 수십 초 걸리는데 피드백이 없어 "실행 중인지" 알 수 없다.
- 현재 대응: 확정된 무피드백을 유지한다. 단, 이 지점은 **위임된 열린 결정**으로 표시(섹션 2). 향후 "실행 중 메뉴바 아이콘 미세 변경" 같은 최소 표시를 합의 후 추가할 수 있다.
- 내부 처리: 무피드백이라도 실행은 **UI를 막지 않는 백그라운드**에서 돌린다 (드롭다운 클릭 즉시 닫히고 앱이 멈추지 않도록).

**8.5 관리 창 열기 (메뉴바 → 별도 창)**
- 문제: 메뉴바 드롭다운에서 독립된 관리 창을 띄우는 흐름이 필요하다.
- 해결: 앱에 별도 창 scene을 두고, 드롭다운의 "명령어 관리…"에서 창 열기 액션을 호출한다. 관리 창과 드롭다운은 같은 `CommandStore` 상태를 공유해 편집이 즉시 반영되게 한다.

**8.6 실행 실패의 조용한 처리**
- 문제: 무피드백이라 실패해도 사용자에게 안 알린다.
- 해결(현재): 종료 코드는 내부적으로만 확인하고 표시하지 않는다. 향후 "실패 시에만 알림" 옵션이 필요해지면 위임된 결정에서 조정.

## 9. 개발 로드맵 (실행·검증 가능한 마일스톤)

각 마일스톤은 **관찰 가능한 동작**으로 검증한다. (바이브코딩: 협업자가 자율 구현하고, 아래 "확인 방법"으로 사용자가 눈으로 확인한다.)

**M0 — 프로젝트 셋업 & 빈 메뉴바 앱**
- 내용: macOS 앱 프로젝트 생성(최소 macOS 13), 샌드박스 OFF, `MenuBarExtra` 뼈대.
- ✅ 확인 방법: 앱을 실행하면 화면 상단 **메뉴바에 아이콘이 뜨고**, 클릭하면 (아직 비어있는) 드롭다운이 열린다.

**M1 — 하드코딩된 한 개 명령 실행 (KeyFlow)**
- 내용: 위 KeyFlow 빌드+실행 스크립트를 코드에 박아 버튼 1개로 실행. `CommandRunner` 초안.
- ✅ 확인 방법: 메뉴바 → 버튼 클릭 → 잠시 후 **KeyFlow가 빌드되어 자동 실행**된다. (드롭다운은 클릭 즉시 닫힌다.)

**M2 — 명령 목록 데이터 모델 + 영속화**
- 내용: `RunCommand` 모델, `CommandStore`, JSON 저장/로드. 드롭다운이 목록을 렌더링.
- ✅ 확인 방법: (코드로 임시 주입한) **여러 명령이 드롭다운에 리스트로** 뜨고, 앱을 껐다 켜도 목록이 유지된다.

**M3 — 관리 창 (추가/수정/삭제)**
- 내용: 별도 관리 창에서 명령 CRUD. 드롭다운 하단 "명령어 관리…" 진입점.
- ✅ 확인 방법: 관리 창에서 **새 명령을 추가하면 드롭다운에 바로 나타나고**, 수정·삭제도 즉시 반영된다. 재실행 후에도 유지.

**M4 — 실행 견고화**
- 내용: 로그인 셸/PATH 확정, 백그라운드 실행으로 UI 비블로킹, 실패 시 내부 로깅(표시는 안 함).
- ✅ 확인 방법: **Homebrew 도구를 쓰는 명령도 성공**하고, 오래 걸리는 명령 실행 중에도 **앱이 멈추지 않는다**(다른 항목 클릭·관리 창 열기 정상).

**M5 — 마감 (선택 항목 포함)**
- 내용: 앱 아이콘, (옵션) 로그인 시 자동 실행, 전체 정리.
- ✅ 확인 방법: 자동 실행 옵션을 켜면 **재부팅 후 메뉴바에 자동으로** 뜬다.

## 10. cowork 협업 규칙 (바이브코딩)

이 프로젝트는 **바이브코딩** 방식이다. 협업자(Claude)는 아래 규약을 따른다.

**행동 규약**
1. **자율 구현.** 사용자에게 코드 이해나 코드 리뷰를 요구하지 않는다. 협업자가 끝까지 완성한다.
2. **보고는 관찰 가능한 동작 중심.** 코드 내부 설명이 아니라 "무엇이 되는지 / 어떻게 켜서 확인하는지 / 어떤 모습인지"로 보고한다.
   - 좋은 보고 예: "메뉴바에 아이콘이 떴습니다. 클릭하면 명령 3개가 리스트로 보이고, 'KeyFlow 빌드+실행'을 누르면 몇 초 뒤 KeyFlow가 실행됩니다. 확인하려면 앱을 실행하고 메뉴바 우측을 보세요."
   - 피할 보고 예: 내부 클래스 구조·함수 시그니처를 늘어놓는 것.
3. **마일스톤 단위 진행.** 각 M을 끝낼 때마다 "이 동작이 되는지 확인해달라"는 형태로 확인 방법을 함께 제시한다.
4. **확정 명세 임의 변경 금지.** 섹션 2의 확정 사항은 바꾸지 않는다. 바꿀 이유가 보이면 **트레이드오프와 함께 합의를 먼저** 받는다.
5. **위임된 결정은 단정하지 않는다.** 합리적 기본값으로 진행하되, 무엇을 왜 골랐는지 보고한다. 자잘한 결정으로 흐름을 끊지 않는다.
6. **추측한 API를 넣지 않는다.** 불확실한 API/시그니처는 코드에 넣기 전 Apple 공식 문서로 확인한다. 버전 한정 API(예: macOS 13+ `MenuBarExtra`, `SMAppService`)는 가용성을 명시한다.
7. **용어는 Swift/SwiftUI/AppKit로.** 설명·예시에 다른 스택 비유를 쓰지 않는다.
8. **가칭 표기.** 정식 명칭이 없으면 가칭을 정해 쓰고 교체 가능함을 밝힌다.

## 11. 부록 — 핵심 API / 빠른 참조

> 아래는 이름과 용도, 가용성만 정리한 참조 목록이다. 실제 시그니처는 구현 시 공식 문서로 확인한다.

**메뉴바 껍데기**
- `MenuBarExtra` (SwiftUI, **macOS 13+**) — 메뉴바에 상주하는 앱 scene을 만든다. 라벨(아이콘/텍스트)과 내용 뷰를 구성.
- `.menuBarExtraStyle(_:)` — 드롭다운 표현 방식. 단순 클릭 목록은 메뉴 스타일, 더 풍부한 커스텀 UI가 필요하면 윈도우(팝오버) 스타일. 초기엔 단순 스타일로 시작.

**별도 창 열기**
- `WindowGroup` / `Window` (SwiftUI scene) — 관리 창 정의.
- `openWindow` 환경 액션 (**macOS 13+**) — 드롭다운에서 관리 창을 여는 데 사용.

**셸 실행**
- `Process` (Foundation) — 외부 프로세스 실행.
  - 실행 파일: `/bin/zsh`
  - 인자: `["-l", "-c", 스크립트문자열]` (로그인 셸 + 명령 실행)
  - 백그라운드로 실행해 UI를 막지 않는다. 종료 코드는 내부 확인용으로만.

**영속화**
- `Codable` + `JSONEncoder`/`JSONDecoder` — `RunCommand` 목록 직렬화.
- `FileManager` — Application Support 디렉토리 경로 확보 및 `commands.json` 읽기/쓰기.

**(선택) 로그인 시 자동 실행 — M5**
- `SMAppService.mainApp` (ServiceManagement, **macOS 13+**) — 앱을 로그인 항목으로 등록/해제.
